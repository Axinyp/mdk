# 代码组织方式

一个程序由以下若干个块组成，**必须严格按顺序排列**：

```
DEFINE_DEVICE      // 设备定义块
DEFINE_COMBINE     // 设备组合块（可选）
DEFINE_CONSTANT    // 常量定义块
DEFINE_VARIABLE    // 变量定义块
DEFINE_FUNCTION    // 函数定义块
DEFINE_TIMER       // 定时器定义块（可选）
DEFINE_START       // 程序初始化语句块
DEFINE_EVENT       // 事件定义块
DEFINE_PROGRAME    // 程序循环语句块（可选）
```

**说明**：
(1) 每个块都有其特定的功能；如"DEFINE_DEVICE"专门用来定义设备，程序里要用到的设备都必须在这个块里进行定义。
(2) 不建议改变各块之间的顺序。
(3) **空块不加大括号 `{ }`**，直接空行或省略。

---

## 各块约束速览

| 块 | 核心约束 | 详细文档 |
|---|---|---|
| `DEFINE_DEVICE` | 格式：`名 = 载体类型:ID:元设备类型;` | [DEFINE_DEVICE.md](DEFINE_DEVICE.md) |
| `DEFINE_COMBINE` | 可选；空块不加 `{ }` | [DEFINE_COMBINE.md](DEFINE_COMBINE.md) |
| `DEFINE_CONSTANT` | ⚠️ **只允许整型**，禁止字符串/浮点 | [DEFINE_CONSTANT.md](DEFINE_CONSTANT.md) |
| `DEFINE_VARIABLE` | ⚠️ **类型小写**（`int`/`string`），**必须赋初值** | [DEFINE_VARIABLE.md](DEFINE_VARIABLE.md) |
| `DEFINE_FUNCTION` | ⚠️ **只能出现一次**；被调用者定义在前 | [DEFINE_FUNCTION.md](DEFINE_FUNCTION.md) |
| `DEFINE_TIMER` | 可选；必须在调用它的函数之前，或放 `DEFINE_EVENT` 中 | [DEFINE_TIMER.md](DEFINE_TIMER.md) |
| `DEFINE_START` | 上电执行一次；可调函数、启动定时器 | [DEFINE_START.md](DEFINE_START.md) |
| `DEFINE_EVENT` | ⚠️ **事件用圆括号**；`DATA.Data` 只在 `ONDATA` 内使用 | [DEFINE_EVENT.md](DEFINE_EVENT.md) |
| `DEFINE_PROGRAME` | 可选；优先用事件驱动，避免阻塞 | [DEFINE_PROGRAME.md](DEFINE_PROGRAME.md) |
