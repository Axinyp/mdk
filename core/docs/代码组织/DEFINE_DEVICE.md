**DEFINE_DEVICE:**  **设备定义块**
**设备定义必须放在此块中。**
**设备定义格式**
设备名=[载体设备类型]:[载体设备ID]:[元设备类型];
**ZigBee设备格式**
设备名=[载体设备类型]:[载体设备ID]:[元设备类型]:[设备SN码];
**1、设备名**
设备名的命名规则与变量的命名规则一样，具体请参考 " [类型和值](1、类型和值.htm)" 里标识符的命名规则。

**2、载体设备类型**
**说明：载体设备类型表示是一个大设备，其上有其他设备。如主机板，就是一个大设备，其上有串口设备、继电器、红外口、io口等。**
**对于触屏，载体设备跟元设备一体。用T表示载体类型，TP表示元设备类型。**

| [设备类型] | 描述           |
| ------ | ------------ |
| M      | 表示Main主机     |
| T      | 表示Touch触摸屏设备 |
| N      | 表示CRNET设备    |
| L      | 表示CRLINK设备   |
| Z      | 表示ZigBee设备   |


**3、载体设备ID**
整型常量，如：1001,2001,3003……
**4、元设备类型**
说明：**元设备表示在载体设备上的小型设备，如串口，灯光，音量等。**
I、设备元素类型

|   |   |
|---|---|
|[设备元素类型]|描述|
|RELAY|表示继电器|
|COM|表示COM口|
|TP|表示触摸屏|
|IR|表示红外|
|IO|表示IO|
|LITE|表示调光器|
|VOL|表示音量控制器|
|WM|墙上面板|

|   |   |
|---|---|
|DMX512|512跑马灯|

II、元设备都带有通道号，其通道号表示对于第几个具体设备。

**5、例子**
```
DEFINE_DEVICE
RELAY   =M:1001:RELAY; // 定义主机板1001上的继电器
COM     =M:1001:COM; // 定义主机板1001上COM口设备
Tp1     =T:1002:TP; // 定义ID为1002的触摸屏
// RELAY2  =S:2001:RELAY; // ⚠️ S类型（级联板）在当前版本中不支持，不要使用
RELAY3  =N:2004:RELAY; //定义CRNET设备2004上的继电器
```
