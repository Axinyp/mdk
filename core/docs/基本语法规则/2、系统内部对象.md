**系统内部对象**
说明：内部对象是针对**DEFINE_EVENT**块里面的按钮事件、拉条事件、数据事件的。表示在触发该事件的时候
通过内部对象可以看到触发该事件的设备的详细信息。特别在事件触发事件中不带设备channel的时候，可以通过内部对象
来检查通道号。eg：
```
BUTTON_EVENT(tp_1)  
 {  
   PUSH()  
   {  
    switch(BUTTON.Channel)  
    {  
     case 1: //第一个按钮按下  
         //操作  
       break;  
     case 2://第二个按钮按下  
      // 操作  
       break;  
   }  
  }  
} 
```
   A、BUTTON

| 属性名        | 类型   | 说明                      |
| ---------- | ---- | ----------------------- |
| DeviceType | char | 设备类型，如：’M’，’S’等         |
| DeviceID   | Int  | 设备ID                    |
| Module     | int  | 设备元素类型,如：’ TP’， ’RELAY’ |
| Channel    | int  | 元素标识号                   |
| JoinNumber | int  | 触发按钮的通道号                |
注：这里 Channel 等同 JoinNumber
  B、LEVEL

| 属性名        | 类型   | 说明                      |
| ---------- | ---- | ----------------------- |
| DeviceType | char | 设备类型，如：’M’，’S’等         |
| DeviceID   | int  | 设备ID                    |
| Module     | int  | 设备元素类型,如：’ TP’， ’RELAY’ |
| Channel    | int  | 元素标识号                   |
| JoinNumber | int  | 拉条的通道号                  |
| Value      | int  | 拉条的值(模拟量)               |

　注：这里 Channel 等同 JoinNumber

eg：通过拉条设置音量设备第一通道的音量值。

```
LEVEL_EVENT(tp_1,2)  
 {  
  SET_VOLTOTOL(vol,1,LEVEL.Value);  
 }
```

 C、DATA

| 属性名        | 类型     | 说明                      |
| ---------- | ------ | ----------------------- |
| DeviceType | char   | 设备类型，如：’M’，’S’等         |
| DeviceID   | int    | 设备ID                    |
| Module     | int    | 设备元素类型,如：’ TP’， ’RELAY’ |
| Channel    | int    | 元素标识号                   |
| Data       | byte[] | DATA事件所附带的数据            |
