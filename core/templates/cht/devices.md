# CHT 设备声明规则

## 声明格式
```
设备名 = [载体类型]:[载体ID]:[元设备类型];
```

## 载体类型（第一段）

| 符号 | 含义 | 典型场景 |
|------|------|---------|
| `T`  | Touch 触摸屏 | 触摸屏，载体和元设备一体 |
| `M`  | Main 主机板 | 主机板自带的 COM/RELAY/IR/IO |
| `L`  | CRLINK 扩展模块 | TR-0740S、TS-9101 等扩展板 |
| `N`  | CRNET 网络设备 | 网络挂载的设备 |

## 有效元设备类型（第三段）

⚠️ 只能使用下列类型，严禁造出不存在的类型：

| 元设备类型 | 描述 |
|-----------|------|
| `TP`      | 触摸屏 |
| `RELAY`   | 继电器 |
| `COM`     | 串口 |
| `IR`      | 红外口 |
| `IO`      | IO口 |
| `LITE`    | 调光器 |
| `VOL`     | 音量控制器 |
| `WM`      | 墙上面板 |
| `DMX512`  | DMX512跑马灯 |

**DSP、MATRIX、CAM 不是元设备类型，不在 DEFINE_DEVICE 声明。**
**TCP/UDP 控制的设备（电脑、投屏、音频处理器）不需要声明设备，直接 SEND_TCP/SEND_UDP 即可。**

## ⚠️ 命名规则（关键）

### 规则一：设备名后缀用顺序号，不用板号

同一型号出现多次时，设备名后缀从 1 开始顺序编号，**与载体 ID 无关**：

```
// 需求：两块 TR-0740S，板号分别是 2 和 3，各有一个 IR 口
TR_0740S_IR1 = L:2:IR;   // ✅ 名字后缀 1（顺序第1个），载体ID=2（真实板号）
TR_0740S_IR2 = L:3:IR;   // ✅ 名字后缀 2（顺序第2个），载体ID=3（真实板号）

// 错误写法：
TR_0740S_IR2 = L:2:IR;   // ❌ 把板号 2 用作名字后缀，容易混淆
TR_0740S_IR3 = L:3:IR;   // ❌ 把板号 3 用作名字后缀
```

### 规则二：设备名以硬件型号为基础

```
// TS-9101 的继电器
TS-9101_RlY = L:1:RELAY;   // ✅ 型号 + 元设备类型缩写

// TR-0740S 的 COM 口（两块板）
TR_0740S_COM1 = L:2:COM;   // ✅ 顺序1，板号2
TR_0740S_COM2 = L:3:COM;   // ✅ 顺序2，板号3

// TR-0740S 的 IR 口（同两块板）
TR_0740S_IR1 = L:2:IR;     // ✅ 顺序1，板号2
TR_0740S_IR2 = L:3:IR;     // ✅ 顺序2，板号3
```

### 规则三：同一物理板的多种功能分开声明

同一块板（如 TR-0740S 板号2）可以同时有 COM 和 IR：
```
TR_0740S_COM1 = L:2:COM;   // 板号2 的串口（控制窗帘）
TR_0740S_IR1  = L:2:IR;    // 板号2 的红外（控制空调）
```
两行都是 `L:2:`，这是正确的——同一块板的不同功能端口。

### 规则四：哪些控制不需要声明设备

- **继电器控制的负载**（灯光、桌插）→ 不单独声明，使用 TS-9101_RlY 的通道
- **UDP/TCP 控制**（电脑切换、投屏、DSP）→ 不声明，直接 `SEND_UDP` / `SEND_TCP`
- **同一 RELAY 设备的多路** → 只声明一次 RELAY，用通道号区分

## 完整示例

需求：TS-9101(板1)控灯，触摸屏10，两块TR-0740S(板2、3)控窗帘(COM/RS485)+空调(IR)，UDP控电脑：

```
DEFINE_DEVICE
    TS-9101_RlY  = L:1:RELAY;   // TS-9101 板1 继电器（控灯光、桌插）
    tp           = T:10:TP;      // 触摸屏 编号10
    TR_0740S_COM1 = L:2:COM;    // TR-0740S 板2 串口（前窗帘 RS485）
    TR_0740S_COM2 = L:3:COM;    // TR-0740S 板3 串口（后窗帘 RS485）
    TR_0740S_IR1  = L:2:IR;     // TR-0740S 板2 红外（空调1）
    TR_0740S_IR2  = L:3:IR;     // TR-0740S 板3 红外（空调2）
    // 电脑/投屏/DSP 不声明设备，用 SEND_UDP/SEND_TCP 直接控制
```

## DEFINE_START 串口初始化

每个 COM 设备必须在 DEFINE_START 中调用 SET_COM：
```
SET_COM(TR_0740S_COM1, 通道, 波特率, 数据位, 校验位, 停止位, 0, 类型);
// 类型: 232=RS232, 485=RS485
// 示例（RS485，9600，8N1）：
SET_COM(TR_0740S_COM1, 2, 9600, 8, 0, 10, 0, 485);
SET_COM(TR_0740S_COM2, 3, 9600, 8, 0, 10, 0, 485);
```
